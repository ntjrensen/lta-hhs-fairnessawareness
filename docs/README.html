Fairness Awareness: Analysis of Equity in Progression of Students in Educational Institutions

This Npuls CEDA project makes it possible to create a fairness analysis to study equity (kansengelijkheid) within progression of students in educational institutions in The Netherlands (mbo, hbo, and wo).

üéì About

This project is a template for creating a fairness analysis of the progression of students in educational institutions in The Netherlands. The template is based on the Quarto framework and uses the Tidyverse coding standard. The template is developed by the Learning Technology & Analytics (LTA) research group of The Hague University of Applied Sciences for the Npuls CEDA programma.

üöÄ Getting started

These instructions will get you a copy of the project up and running on your local machine for development and testing purposes. See the deployment for notes on how to deploy the project on a live system.

Prerequisites

To start working with this project, make sure you have the following software installed following the links provided:

R, Rstudio, Quarto, XQuartz (for Mac only)

  • RStudio version 2024.12.0 or higher with R version 4.4.2 or higher - Posit
  • XQuartz 2.8.5 or higher (for Mac only) - XQuartz
  • Quarto version 1.6.39 or higher - Quarto

Github, Github Desktop

  • A Github account and (optionally) GitHub Desktop version 3.4.16 - Github

R packages, fonts

  • All other packages and fonts are installed automatically by running the _Setup.R file of the project on your local machine. You will find instructions below.

Installing

  1. First make a fork of the repository to your own Github account. Then clone the repository to your local machine, using the following commands or GitHub Desktop. See Cloning a repository for an explanation.
  2. Once you have cloned the repository, open the project in RStudio and run the _Setup.R file by clicking on ‚ÄòSource‚Äô. This will install all the necessary packages and fonts.

üìö Rendering your first fairness reports

Render the _basic project with the default settings

Run the _basic project with the default settings.

  • You will create a report for the The Hague University of Applied Sciences (THUAS) with the default settings. The report is based on synthesized data of the bachelor Communication & Multimedia Design (CMD), and will be created in the _basic folder.
  • The first time you create this report, it will take several minutes.
  • Once the rendering has finished, click on _basic/index.html and choose View in Web Browser. The report will open in your default browser.
  • If you don‚Äôt see this file, click the refresh icon on the top right. If you still don‚Äôt see the report, check the console for any errors.
  • When you rerun the report, the output will be removed and then a new report will be created. If you want to keep the output, copy the folder to a location outside of you project.

Follow these steps:

Since the template has several dependencies, you will have to render the pages using the terminal.

  1. Open a terminal in RStudio (second tab at the bottom of your screen).
  2. Run the following command for the basic report:
quarto render --profile basic to render the basic report.

Render the _advanced project with the default settings

Next, run the _advanced project with the default settings.

  • This time, you will create an extended report. It contains more detailed information about the fairness analysis: prediction models, R-code and an explanation of each step with references.
  • You will create the report on synthesized data of the bachelor Communication & Multimedia Design (CMD) in the _advanced folder.
  • Once it has finished, click on _advanced/index.html and choose View in Web Browser.
  • The first time you create this report, it will take several minutes. The report will open in your default browser.
  • If you don‚Äôt see this file, click the refresh icon on the top right. If you still don‚Äôt see the report, check the console for any errors.
  • When you rerun the report, the output will be removed and then a new report will be created. If you want to keep the output, copy the folder to a location outside of you project.

Follow these steps:

Since the template has several dependencies, you will have to render the pages using the terminal.

  1. Open a terminal in RStudio (second tab at the bottom of your screen).
  2. Run the following command for the advanced report:
quarto render --profile advanced to render the advanced report.

Create and render your own reports

Finally, you can create your own report by changing the settings in the _quarto.yml file.

  • You can change the settings for the layout, the educational institution, faculty or academy, the study program, the version of the report, the success factors, the model, the use of synthetic data, the recreation of plots, the enrollment selection and the includes.
  • Start small and work your way up to more complex settings.
  • Follow the Tutorial to learn you to create and render your own reports.

üèÅ Deployment

The reports are created in the _basic and advanced folders. You can deploy the reports to a web server or share them with others by copying the folder to a location outside of your project. You can also share the reports by uploading them to a web server or by using a service like GitHub Pages.

üî® Built with

  • Quarto - The web framework used in combination with R.
  • Tidyverse - The R coding standard used.
  • Tidymodels - The R package used for building the prediction models.
  • Dalex - The R package used for explaining the prediction models and creating the fairness analysis. The implementation of the fairness analysis is based on the Fairness package.

The Dalex implementation is adjusted by the LTA-HHs team to improve usability and make it suitable for the use in the educational sector in The Netherlands.
